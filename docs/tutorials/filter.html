filter

Returns a lazy iterable of the elements a predicate returns true for.

Iterable<A> filter<A>(bool Function(A a) f, Iterable<A> iterable) FxAsyncIterable<A> filterAsync<A>(FutureOr<bool> Function(A a) f, FxAsyncIterable<A> iterable) Fx<T> Fx.filter(bool Function(T a) f) // chain (alias: where) FxAsync<T> FxAsync.filter(FutureOr<bool> Function(T a) f)

Lecture

filter is the other fundamental transformer alongside map: it keeps every element for which the predicate returns true and drops the rest, lazily. Nothing runs until a terminal operator pulls values through, and since Fx extends Iterable, .where(...) works too — it's just an alias for .filter(...) on the chain.

Because it's lazy, combining filter with take only evaluates as many predicate calls as needed to satisfy the take — see Demo 1.

filterAsync has its own dedicated concurrent implementation (rather than reusing mapAsync's): when you attach .concurrent(n), n predicates are evaluated in parallel, but the elements that pass are still released downstream in their original order — concurrency changes throughput, never the result.
