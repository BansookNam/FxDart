toArray

Materializes a lazy iterable — pulling every value and collecting them into a List.

List<A> toArray<A>(Iterable<A> iterable) Future<List<A>> toArrayAsync<A>(FxAsyncIterable<A> iterable) List<T> Fx.toArray() // chain Future<List<T>> FxAsync.toArray()

Lecture

toArray is the workhorse terminal operator: it's what actually runs a chain that was, until this point, nothing but a plan. Calling .toArray() pulls every remaining value out of the iterable, in order, and collects them into a real List<T>. Everything upstream — every .map(), .filter(), .take() — only runs because toArray asked for values.

That also means toArray is exactly the operator you must not call directly on an infinite or unbounded source (range with no end, cycle, repeat with a huge count) — it will try to pull forever. Bound it first with take(n), then call toArray on the bounded result.

The async version, toArrayAsync (or .toArray() on an FxAsync chain), awaits each element as it's pulled and returns a Future<List<T>>. Combined with .concurrent(n) upstream, the individual awaits can overlap even though the final list still comes back in the original order.
